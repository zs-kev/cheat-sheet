Main Root

Tag Name Description
<html>...</html> HTML Represents the root (top-level element) of an HTML document, so it is also referred to as the root element. All other elements must be descendants of this element.

Document Metadata

Tag Name Description
<head>...</head> Head Contains machine-readable information (metadata) about the document, like its title, scripts, and style sheets.
<link>...</link> External Resource Link Specifies relationships between the current document and an external resource.
<meta> MetaData Represents metadata that cannot be represented by other HTML meta-related elements, like <base>, <link>, <script>, <style> or <title>
<style>...</style> Style Contains style information for a document, or part of a document.
<title>...</title> Title Defines the document’s title that is shown in a browser’s title bar or a page’s tab.

Sectioning Root

Tag Name Description
<body>...</body> Body Represents the content of an HTML document. There can be only one <body> element in a document.

Content Sectioning

Tag Name Description
<header>...</header> Header Represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.
<nav>...</nav> Navigation Represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents, and indexes.
<main>...</main> Main Represents the dominant content of the <body> of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.
<div>...</div> Division A generic container for flow content. It has no effect on the content or layout until styled using CSS.
<section>...</section> Section Represents a standalone section — which doesn't have a more specific semantic element to represent it — contained within an HTML document.
<article>...</article> Article Represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable.
<aside>...</aside> Aside A portion of a document whose content is only indirectly related to the document's main content.
<address>...</address> Address Indicates that the enclosed HTML provides contact information for a person or people, or for an organization.
<footer>...</footer> Footer Represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of the section, copyright data or links to related documents.

Text Content

Tag Name Description
<h1>-<h6>...</h6> Headings represent six levels of section headings. <h1> is the highest section level and <h6> is the lowest.
<p>...</p> Paragraph Represents a paragraph.
<ol>...</ol> Ordered List An ordered list of items, typically rendered as a numbered list.
<ul>...</ul> Un-ordered List An unordered list of items, typically rendered as a bulleted list.
<li>...</li> List Item Used to represent an item in a list.
<figure>...</figure> Figure Represents self-contained content, potentially with an optional caption.
<figcaption>...</figcaption> Figure Caption Represents a caption or legend describing the rest of the contents of its parent
<blockquote>...</blockquote> Block Quotation Indicates that the enclosed text is an extended quotation. Usually, this is rendered visually by indentation. A URL for the source of the quotation may be given.
<hr> Horizontal Rule Represents a thematic break between paragraph-level elements: for example, a change of scene in a story, or a shift of topic within a section.
<pre>...</pre> Preformatted Text Represents preformatted text which is to be presented exactly as written in the HTML file.

Inline Text Semantics

Tag Name Description
<a>...</a> Anchor Creates a hyperlink to web pages, files, email addresses, locations in the same page, or anything else a URL can address.
<span>...</span> Span A generic inline container for phrasing content, which does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang.
<br> Break Produces a line break in text (carriage-return).
<em>...</em> Emphasis Marks text that has stress emphasis. It can be nested, with each level of nesting indicating a greater degree of emphasis.
<small>...</small> Small Represents side-comments and small print, like copyright and legal text, independent of its styled presentation. By default, it renders text within it one font-size small, such as from small to x-small.
<strong>...</strong> Strong Importance Indicates that its contents have strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.
<mark>...</mark> Mark Represents text which is marked or highlighted for reference or notation purposes, due to the marked passage's relevance or importance in the enclosing context.
<time>...</time> Time Represents a specific period in time.
<code>...</code> Code Displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code.
<cite>...</cite> Citation Used to describe a reference to a cited creative work, and must include the title of that work.
<abbr>...</abbr> Abbreviation Represents an abbreviation or acronym.

Images and Multimedia

Tag Name Description
<audio>...</audio> Audio Used to embed sound content in documents.
<video>...</video> Video Embeds a media player which supports video playback into the document.
<img> Image Embeds an image into the document.
<track> Time Tracks Used as a child of the media elements <audio> and <video>. It lets you specify timed text tracks (or time-based data), for example to automatically handle subtitles. The tracks are formatted in WebVTT format (.vtt files) — Web Video Text Tracks or Timed Text Markup Language (TTML).

Scripting

Tag Name Description
<script>...</script> Script Used to embed or reference executable code; this is typically used to embed or refer to JavaScript code.

Table Content

Tag Name Description
<table>...</table> Table Represents tabular data — that is, information presented in a two-dimensional table comprised of rows and columns of cells containing data.
<caption>...</caption> Caption specifies the caption (or title) of a table, and if used is always the first child of a <table>.
<thead>...</thead> Table Head Defines a set of rows defining the head of the columns of the table.
<tbody>...</tbody> Table Body Encapsulates a set of table rows, indicating that they comprise the body of the table.
<tfoot>...</tfoot> Table Foot Defines a set of rows summarizing the columns of the table.
<th>...</th> Table Header Defines a cell as header of a group of table cells.
<tr>...</tr> Table Row Defines a row of cells in a table.
<td>...</td> Table Data Defines a cell of a table that contains data.

Forms

Tag Name Description
<form>...</form> Form Represents a document section that contains interactive controls for submitting information to a web server.
<input> Input Used to create interactive controls for web-based forms in order to accept data from the user; a wide variety of types of input data and control widgets are available, depending on the device and user agent.
<button>...</button> Button A clickable button, which can be used in forms or anywhere in a document that needs simple, standard button functionality.
<datalist>...</datalist> Data List contains a set of <option> elements that represent the values available for other controls.
<select>...</select> Selection Represents a control that provides a menu of options.
<optgroup>...</optgroup> Group Option Creates a grouping of options within a <select> element.
<option>...</option> Option Can represent menu items in popups and other lists of items in an HTML document.
<textarea>...</textarea> Text Area Represents a multi-line plain-text editing control, useful when you want to allow users to enter a sizeable amount of free-form text, for example a comment on a review or feedback form.
<progress>...</progress> Progress Displays an indicator showing the completion progress of a task, typically displayed as a progress bar.
<fieldset>...</fieldset> Field Set Used to group several controls as well as labels within a web form.
<label>...</label> Label Represents a caption for an item in a user interface.
<legend>...</legend> Legend Represents a caption for the content of its parent <fieldset>.

Interactive Elements

Tag Name Description
<details>...</details> Details Creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state.
<summary>...</summary> Summary Specifies a summary, caption, or legend for a <details> element's disclosure box.
